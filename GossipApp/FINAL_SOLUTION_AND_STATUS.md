## ❌ CRITICAL ISSUE - AsyncStorage in Release Builds

### The Problem
**AsyncStorage native module is not being linked in release builds**, causing the app to crash immediately on startup from Play Store with:

```
Error: [@RNC/AsyncStorage]: NativeModule: AsyncStorage is null.
```

### Root Cause
React Native 0.73.6's **autolinking system is not working** for this project:

1. ❌ `PackageList` class is not being auto-generated by React Gradle Plugin
2. ❌ Manual linking causes compilation errors (dependency conflicts)
3. ❌ Debug builds work because Metro bundler links modules at runtime
4. ❌ Release builds fail because modules must be compiled into the APK

5. ✅ Disabled ProGuard → Still failed
6. ✅ Set `debuggable=true` → Still failed

## 💡 WHY THIS IS HAPPENING

React Native's autolinking system has evolved significantly:

- **React Native 0.60-0.62:** Used native_modules.gradle (manual)
- **React Native 0.63-0.72:** Hybrid autolinking
- **React Native 0.73+:** React Gradle Plugin handles everything

**Your GossipApp appears to be in a transitional state** where the React Gradle Plugin isn't properly generating the `PackageList` class needed for autolinking.
